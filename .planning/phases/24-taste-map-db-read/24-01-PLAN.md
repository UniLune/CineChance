# PLAN: Phase 24 - Taste Map DB Read Fix

## Phase Goal
Ensure `/profile/taste-map` displays favorite actors and directors from the `PersonProfile` database table, matching the data shown on `/profile/actors` and `/profile/creators`.

## Tasks Breakdown

### Task 1: Update TasteMapClient Props and Rendering
- **File**: `src/app/profile/taste-map/TasteMapClient.tsx`
- **Changes**:
  - Add `topActors` and `topDirectors` to props interface
  - Remove `useEffect` hooks that fetch from `/api/user/person-profile`
  - Remove fallback fetch to `/api/user/achiev_*`
  - Remove loading states (data comes from server)
  - Render `topActors` and `topDirectors` directly from props
- **Acceptance**: Component receives data via props and displays correctly without client-side fetching

### Task 2: Update Page Server Component
- **File**: `src/app/profile/taste-map/page.tsx`
- **Changes**:
  - Import `prisma` from `@/lib/prisma`
  - Import `PersonData` type from `@/lib/taste-map/person-profile-v2`
  - Fetch `PersonProfile` records for user (actor + director) using `prisma.personProfile.findUnique`
  - Transform data to `Array<[name, score]>` format
  - Pass transformed data to `TasteMapClient` as props
- **Acceptance**: Page passes correct props to client component based on DB content

### Task 3: Type Safety and Error Handling
- **File**: Both files
- **Changes**:
  - Ensure proper TypeScript types (avoid `any`)
  - Add fallback for empty profiles (`|| []`)
  - Slice to top 10 items
- **Acceptance**: TypeScript compiles with zero errors, handles empty state gracefully

### Task 4: Verification
