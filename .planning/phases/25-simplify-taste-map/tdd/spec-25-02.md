# Unit Spec: 25-02 Remove Person Profile UI

## Overview
Remove all person profile (actors/directors) UI components and API fields from TasteMap feature. Backend computation of `personOverlap` remains for DB compatibility but is not displayed.

## Files to Modify

### 1. `src/app/profile/taste-map/page.tsx`
**Current behavior**: Fetches `PersonProfile` data and passes `topActors`/`topDirectors` to `TasteMapClient`.

**Changes**:
- Remove imports: `computeUserPersonProfile`, `PersonData`
- Remove `Promise.all` block that computes person profiles (lines ~25-28)
- Remove Prisma queries for `PersonProfile` (lines ~31-38)
- Remove transformations `topActors` and `topDirectors` (lines ~41-47)
- Remove `topActors` and `topDirectors` props from `<TasteMapClient ... />` (lines ~59-60)

**Expected result**: Server component only fetches taste map data and passes only necessary props (no person data).

### 2. `src/app/profile/taste-map/TasteMapClient.tsx`
**Current behavior**: Renders sections for Top Actors and Top Directors.

**Changes**:
- Remove `topActors?: PersonData[]` and `topDirectors?: PersonData[]` from props interface (lines ~10-11)
- Remove these props from component function parameters (line ~23)
- Remove entire `topActors` section JSX (lines ~168-186)
- Remove entire `topDirectors` section JSX (lines ~188-206)
- Remove any related constants or helper functions for these sections

**Expected result**: Client component renders without actors/directors sections. All other sections remain.

### 3. `src/app/profile/taste-map/TwinTasters.tsx`
**Current behavior**: Tooltip shows weights: 50% movies, 30% genres, 20% persons.

**Changes**:
- Update the explanation text (lines ~136-143) to:
  - 🎬 Совпадение по фильмам (40%)
  - 🎭 Жанры (60%)
  - Remove the 👥 Персоны line completely

**Expected result**: Tooltip accurately reflects new weights.

### 4. `src/app/api/user/taste-map-comparison/[userId]/route.ts`
**Current behavior**: API response includes `personComparison` field computed via `comparePersonProfiles`.

**Changes**:
- Remove import of `comparePersonProfiles` (line ~11)
- Remove call to `comparePersonProfiles` (lines ~152-156)
- Remove `personComparison` property from returned JSON (line ~173)
- Optionally keep `personOverlap` in `metrics` (for backward compatibility) but do not add new `personComparison` object

**Expected result**: API returns JSON with `genreComparison`, `ratingPatterns`, `metrics` (no `personComparison`).

### 5. `src/app/profile/taste-map/compare/[userId]/page.tsx`
**Current behavior**: Renders a detailed person comparison section (actors/directors) when `personComparison` data is available.

**Changes**:
- Remove `PersonOverlapDetail` interface (lines ~41-46)
- Remove `PersonSet` interface (lines ~48-53)
- Remove `PersonComparisonResult` interface (lines ~55-59)
- Remove `personComparison` from `ComparisonData` interface (line ~70)
- Remove entire conditional rendering block for `personComparison` (lines ~630-729)
- Keep `personOverlap` in `MatchMetrics` interface (for compatibility) but do not render a section for it

**Expected result**: Comparison page loads without displaying any person comparison UI.
