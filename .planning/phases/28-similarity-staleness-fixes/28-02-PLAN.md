# Phase 28, Plan 02: Rewrite similar-users API — Cache-First & Staleness-Aware

## Overview

Rewrite the `GET /api/user/similar-users` endpoint to use a three-tier approach:
1. Redis cache (fast path)
2. DB with per-pair freshness check + lazy recompute
3. Full on-demand fallback (if no DB records)

## Current State

The current implementation:
- Reads from DB first (`findMany`)
- If empty → on-demand fallback
- **No Redis cache** in the API (only inside `findSimilarUsers`)
- **No freshness check** (returns stale scores forever)
- Has `freshOnly` parameter that filters post-read (unused)

## New Architecture

```
Request
  ↓
1. Redis cache (getSimilarUsers)
   ├─ HIT → Enrich → Return (fast)
   └─ MISS
       ↓
2. DB with freshness
   ├─ Scores found
   │   ├─ Fresh (expiresAt > now) → use as-is
   │   └─ Stale → lazy recompute → update DB
   └─ No scores
       ↓
3. Full on-demand fallback
   (existing logic, but stores to Redis at end)
       ↓
4. Store to Redis (storeSimilarUsers)
       ↓
Return enriched results
```

### Task 2: Rewrite DB reading with freshness check

**File:** `src/app/api/user/similar-users/route.ts`

Replace the current DB query block (around lines 82-169) with:

```typescript
// Query SimilarityScore from DB with freshness awareness
const scores = await prisma.similarityScore.findMany({
  where: {
    OR: [
      { userIdA: userId },
      { userIdB: userId },
    ],
  },
  orderBy: { overallMatch: 'desc' },
  take: limit,
});

if (scores.length === 0) {
  // CASE: No scores in DB → full on-demand computation
  logger.info('No similarity scores found, computing on-demand', {
    userId,
    context: 'SimilarUsersAPI',
  });

  const candidateIds = await getCandidateUsersForSimilarity(userId);
  const computedScores: typeof scores = [];

  for (const candidateId of candidateIds) {
    try {
      const result = await computeSimilarity(userId, candidateId, false);

      if (isSimilar(result)) {
        await computeAndStoreSimilarityScore(userId, candidateId, 'on-demand');

        const stored = await prisma.similarityScore.findUnique({
          where: {
            userIdA_userIdB: userId < candidateId
              ? { userIdA: userId, userIdB: candidateId }
              : { userIdA: candidateId, userIdB: userId },
          },
        });

        if (stored) {
          computedScores.push(stored);
        }
      }
    } catch (err) {
      logger.debug('Error computing similarity', {
        candidateId,
        error: err instanceof Error ? err.message : String(err),
        context: 'SimilarUsersAPI',
      });
    }
  }

  // Enrich and cache
  const enrichedResults = await enrichSimilarUsers(computedScores, userId);

  // Store to Redis for fast subsequent reads
  const redisScores = enrichedResults.map(u => ({
    userId: u.userId,
    overallMatch: u.overallMatch / 100,
  }));
  await storeSimilarUsers(userId, redisScores);

  return NextResponse.json({
    similarUsers: enrichedResults,
    fromDatabase: false,
    computedAt: new Date().toISOString(),
    message: `Computed ${enrichedResults.length} similar user(s) on-demand`,
  });
} else {
  // CASE: Has scores in DB → check freshness per-pair
  logger.info('Found existing scores, checking freshness', {
    userId,
    totalScores: scores.length,
    context: 'SimilarUsersAPI',
  });

  const freshScores: typeof scores = [];
  const stalePairs: Array<{ score: typeof scores[0]; otherUserId: string }> = [];

  for (const score of scores) {
    const isFresh = !score.expiresAt || score.expiresAt > new Date();

    if (isFresh) {
      freshScores.push(score);
    } else {
      const otherUserId = score.userIdA === userId ? score.userIdB : score.userIdA;
      stalePairs.push({ score, otherUserId });
    }
  }

  // Lazy recompute stale pairs
  for (const { score, otherUserId } of stalePairs) {
    try {
      logger.debug('Recomputing stale similarity pair', {
        userId,
        otherUserId,
        oldScore: Number(score.overallMatch),
        context: 'SimilarUsersAPI',
      });

      const result = await computeSimilarity(userId, otherUserId, false);

      if (isSimilar(result)) {
        await computeAndStoreSimilarityScore(userId, otherUserId, 'on-demand');

        const refreshed = await prisma.similarityScore.findUnique({
          where: {
            userIdA_userIdB: score.userIdA < score.userIdB
              ? { userIdA: score.userIdA, userIdB: score.userIdB }
              : { userIdA: score.userIdB, userIdB: score.userIdA },
          },
        });

        if (refreshed) {
          freshScores.push(refreshed);
        }
      }
      // If not similar anymore → old score remains (will expire via TTL)
    } catch (err) {
      logger.debug('Error recomputing similarity pair', {
        userId,
        otherUserId,
        error: err instanceof Error ? err.message : String(err),
        context: 'SimilarUsersAPI',
      });
      // Keep old score if recompute fails
      freshScores.push(score);
    }
  }

  // Sort by overallMatch descending (in case new scores were added)
  freshScores.sort((a, b) => Number(b.overallMatch) - Number(a.overallMatch));

  // Enrich and cache
  const enrichedResults = await enrichSimilarUsers(freshScores, userId);

  // Store to Redis for fast subsequent reads
  const redisScores = enrichedResults.map(u => ({
    userId: u.userId,
    overallMatch: u.overallMatch / 100,
  }));
  await storeSimilarUsers(userId, redisScores);

  return NextResponse.json({
    similarUsers: enrichedResults,
    fromDatabase: true,
    computedAt: new Date().toISOString(),
    message: `Found ${enrichedResults.length} similar user(s) (${stalePairs.length} refreshed)`,
  });
}
```

**Note:** You may need to create an `enrichSimilarUsers()` helper function that:
1. Extracts other user IDs from scores
2. Fetches user info from DB
3. Gets completed watch counts via `getUserCompletedWatchCount()`
4. Filters by ≥40% and ≥3 movies
5. Returns enriched array
