# Phase 28, Plan 03: Invalidation Triggers & Scheduler Fixes

## Overview

Add cache invalidation for similarity scores when users change their ratings/status. Also fix scheduler thresholds to include new users.

## Tasks

### Task 1: Add invalidation to watchlist/route.ts — POST handler

**File:** `src/app/api/watchlist/route.ts`

Add imports at the top (after existing imports):
```typescript
import { invalidateTasteMap } from '@/lib/taste-map/redis';
import { deleteSimilarityScoresByUser } from '@/lib/taste-map/similarity-storage';
```

In the POST handler, after the successful upsert (around line 493: `await invalidateUserCache(session.user.id)`), add:

```typescript
// Invalidate taste map cache and similarity scores
await invalidateTasteMap(session.user.id);
await deleteSimilarityScoresByUser(session.user.id);
```

This should be added in TWO places within POST handler:
1. After `isRatingOnly` block (around line 237: `return NextResponse.json({ success: true });`)
2. After `isRewatch` block (around line 360: `return NextResponse.json({ success: true });`)
3. After the main status change upsert (around line 551: `return NextResponse.json({ success: true, record });`)

**Note:** All three blocks already call `after(async () => { await recomputeTasteMap(...) })`. Add the invalidation BEFORE the `after()` call or as part of the same block — but BEFORE `return`, not inside `after()`.
